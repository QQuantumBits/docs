## Overview

The `/worlds` command provides administrative functions for managing the LilWorlds plugin itself, including configuration reloading, system maintenance, and global operations.

<Info>
**Admin Only:** Most `/worlds` commands require administrative permissions and should be used carefully.
</Info>

## Plugin Management

### Reload Commands

<CodeGroup>
```bash Complete Reload
# Reload all configurations and components
/worlds reload
/worlds rl

# Reload everything including worlds
/worlds reload all
```

```bash Specific Reloads
# Reload main configuration
/worlds reload config

# Reload world configurations
/worlds reload worlds

# Reload custom generators
/worlds reload generators

# Reload message files
/worlds reload messages
```
</CodeGroup>

### Configuration Management


## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Reload Fails">
    **Symptoms:** Configuration reload commands fail
    
    **Solutions:**
    - Check configuration file syntax
    - Verify file permissions
    - Use `/worlds config validate` first
    - Check console for specific errors
  </Accordion>
  
  <Accordion title="Performance Degradation">
    **Symptoms:** Server lag or slow responses
    
    **Solutions:**
    - Run `/worlds performance` to identify issues
    - Use `/worlds optimize --memory`
    - Check for memory leaks with `/worlds memory`
    - Consider unloading unused worlds
  </Accordion>
  
  <Accordion title="Integration Problems">
    **Symptoms:** Other plugins not working with LilWorlds
    
    **Solutions:**
    - Check `/worlds integrations list`
    - Reload specific integration
    - Verify plugin compatibility
    - Check for version conflicts
  </Accordion>
</AccordionGroup>
